# SynthFrame Wireframe Knowledge Base

This document defines the component specification for Athena AI. Use these EXACT types and property structures.

## Component Types (UPPERCASE)

| Type | Props | Description |
|------|-------|-------------|
| NAVIGATION-BAR | `{"logo": string, "items": string[], "cta": string}` | Top navigation bar |
| HERO-BANNER | `{"headline": string, "subheadline": string, "cta": string}` | Large banner section |
| FEATURE-GRID | `{"features": [{"title", "description", "icon"}]}` | Grid of feature cards |
| CONTENT-BLOCK | `{"title": string, "content": string, "imagePosition": "left"|"right"}` | Text + Image block |
| GALLERY-GRID | `{"images": string[]}` | Matrix of image placeholders |
| TESTIMONIAL-SLIDER | `{"testimonials": [{"name", "role", "quote"}]}` | Customer reviews slider |
| PRICING-TABLE | `{"plans": [{"name", "price", "features"}]}` | Pricing tiers comparison |
| CALL-TO-ACTION | `{"headline": string, "buttonText": string}` | Final nudge to convert |
| FOOTER-SIMPLE | `{"copyright": string, "links": string[]}` | Bottom legal/links section |
| SIDEBAR | `{"items": string[]}` | Vertical side menu |
| BOTTOM_NAV | `{"items": string[]}` | Mobile bottom navigation |
| FORM | `{"fields": [{"label", "type", "placeholder"}], "submit": string}` | Input collection form |
| BUTTON | `{"label": string, "variant": "primary"|"secondary"}` | Clickable action |
| INPUT | `{"placeholder": string, "type": string}` | Single data entry field |
| TEXT | `{"content": string}` | Long-form reading text |
| HEADING | `{"text": string, "level": 1-6}` | Visual hierarchy title |
| IMAGE | `{"alt": string, "src": string}` | Graphic element placeholder |
| TABLE | `{"columns": string[], "rows": number}` | Structured data display |
| CALENDAR | `{"view": "month"|"week"}` | Date picking/viewing widget |
| CHART | `{"type": "bar"|"line"|"pie", "title": string}` | Data visualization |
| FRAME | `{"device": string, "width": number, "height": number}` | Browser/Phone container |

## Rules
1. Component types MUST be UPPERCASE and use dashes (e.g., "HERO-BANNER").
2. Standard layouts should follow: NAVIGATION-BAR -> HERO-BANNER -> CONTENT-BLOCK -> FEATURE-GRID -> FOOTER-SIMPLE.
3. For mobile, use BOTTOM_NAV and single-column layouts.
4. Every component needs a unique `id` and appropriate `size`/`position`.

## Component Structure

Each component has this structure:

```json
{
  "id": "comp-1",
  "type": "NAVBAR",
  "position": {"x": 0, "y": 0},
  "size": {"width": 1440, "height": 64},
  "props": {"logo": "Brand", "links": ["Home", "About"], "cta": "Login"},
  "children": [],
  "source": "llm"
}
```

## Canvas Sizes by Device

- **macbook**: 1440 × 900
- **iphone**: 393 × 852

## Layout Patterns

**Landing Page**: NAVBAR -> HERO -> SECTION/CARD grid -> FOOTER
**Dashboard**: NAVBAR + SIDEBAR -> HEADING -> CARD stats -> TABLE/CHART -> FOOTER
**Mobile App**: NAVBAR -> content sections -> BOTTOM_NAV

## Rules

1. Component types are UPPERCASE ("NAVBAR" not "navbar").
2. All positions/sizes are in pixels (integers).
3. Components must fit within canvas dimensions.
4. Every component needs a unique id (e.g., "nav-1", "btn-submit").
5. source is "llm" for AI-generated, "cv" for sketch-detected.
6. children array is empty [] for now.
